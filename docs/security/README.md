# Security artifacts

This directory holds artifacts for security reviews — including the Zoom Marketplace submission.

## What lives here

- `dast-reports/` — HTML/JSON output from OWASP ZAP baseline scans of the deployed app. Generated by `scripts/dast-scan.sh`.
- `ssdlc.md` — internal-facing notes on our SSDLC (the customer-facing summary lives at `/security` in the app).

## How to produce evidence for a submission

### SAST results

Static analysis is automated via GitHub Actions:

- `.github/workflows/codeql.yml` runs CodeQL on every push, every PR, and weekly.
- Results appear under the repository's **Security → Code scanning** tab.

To capture evidence:

1. Go to the GitHub repo → **Security** → **Code scanning**.
2. Filter by `Tool = CodeQL` and `Branch = main`.
3. Export the latest finding list (or screenshot the dashboard).

### Dependency scan results

- `.github/workflows/dependency-audit.yml` runs `npm audit` daily and on every PR.
- Dependabot is enabled (`.github/dependabot.yml`) — view alerts under **Security → Dependabot**.

To capture evidence:

```bash
npm audit --json > docs/security/dependency-audit-$(date -u +%Y%m%d).json
```

### DAST results

```bash
# Production scan (passive only, safe)
scripts/dast-scan.sh

# Or a specific environment
scripts/dast-scan.sh https://staging.qualitative.one
```

The script runs OWASP ZAP's baseline scanner in Docker. Output lands in `docs/security/dast-reports/zap-YYYYMMDD-HHMMSS.html`. Hand that file to Zoom.

> Baseline = passive only (no attack traffic). Never run `zap-full-scan.py` against production.

### TLS verification

```bash
# Confirms TLS 1.2+ and HSTS header
curl -sIv https://qualitative.one 2>&1 | grep -iE "TLSv1|strict-transport-security"
```

Or use the SSL Labs report at https://www.ssllabs.com/ssltest/analyze.html?d=qualitative.one — submit the resulting grade alongside the Zoom application.
